# 什么是函数柯里化

在维基百科中，柯里化的定义是：在计算机科学中，柯里化（英语：Currying），又译为卡瑞化或加里化，是把接受多个参数的函数变换成接受一个单一参数（最初函数的第一个参数）的函数，并且返回接受余下的参数的新函数的技术。

听起来有些绕，用式子表达就是：能将一个调用形式为f(a, b, c)的函数转化调用形式为f_curry(a)(b)(c)的技术。 比如

function add(a, b) {
    return a + b;
}
add(1, 2) // 3

function add_curry (a) {
    return function(b) {
        return a + b;
    } 
}
add_curry(1)(2)

以上，add_curry是add的柯里化函数，而将add转换为add_curry的技术叫做函数柯里化。

因此，函数柯里化的任务就是将add转换为add_curry，而这也是通过一个函数来实现，在本文中，我们暂时称为createCurry。

# 函数柯里化的用途

在将如何实现函数柯里化之前，为了更好的理解，我们先讲讲它的用途。

举个例子(直接采用「前端进阶」彻底弄懂函数柯里化中的例子)： 在工作中我们经常需要对一些值进行正则校验，我们可能会写一个通用函数

function checkByRegExp(regExp,string) {
    return regExp.test(string);  
}
checkByRegExp(/^1\d{10}$/, '18642838455'); // 校验电话号码
checkByRegExp(/^(\w)+(\.\w+)*@(\w)+((\.\w+)+)$/, 'test@163.com'); // 校验邮箱

大部分情况下使用好像没什么问题，但是如果我们同时要校验一堆电话号码，我们可能就会这样

checkByRegExp(/^1\d{10}$/, '18642838455'); // 校验电话号码
checkByRegExp(/^1\d{10}$/, '13109840560'); // 校验电话号码
checkByRegExp(/^1\d{10}$/, '13204061212'); // 校验电话号码

checkByRegExp(/^(\w)+(\.\w+)*@(\w)+((\.\w+)+)$/, 'test@163.com'); // 校验邮箱
checkByRegExp(/^(\w)+(\.\w+)*@(\w)+((\.\w+)+)$/, 'test@qq.com'); // 校验邮箱
checkByRegExp(/^(\w)+(\.\w+)*@(\w)+((\.\w+)+)$/, 'test@gmail.com'); // 校验邮箱

也就是即使是校验同一类型的值时，我们还要将正则表达式每次都重复一遍，有没有什么办法可以偷懒呢？

const checkByRegExp_curry = createCurry(checkByRegExp)
const checkPhone = checkByRegExp_curry(/^1\d{10}$/)
const checkEmail = checkByRegExp_curry(/^(\w)+(\.\w+)*@(\w)+((\.\w+)+)$/)

checkCellPhone('18642838455'); // 校验电话号码
checkCellPhone('13109840560'); // 校验电话号码
checkCellPhone('13204061212'); // 校验电话号码

checkEmail('test@163.com'); // 校验邮箱
checkEmail('test@qq.com'); // 校验邮箱
checkEmail('test@gmail.com'); // 校验邮箱

这样对于不同的正则类型，我们都有了专门的函数，简单又明了。

柯里化的这种用途可以被称为：参数复用。本质上，它是降低了原函数checkByRegExp的通用性，实现适用性checkCellPhone的提升。

以上的将checkByRegExp转为checkByRegExp_curry的过程是函数柯里化，而checkCellPhone和checkEmail实际上是偏函数的应用——创造了一个新函数，同时将部分参数替换成特定值。不过我觉得没必要一定要严格区分，柯里化技术的一部分本来也是会返回一个偏函数（后面的实现会讲到）

# 函数柯里化的实现

我们先来明确下我们的问题：我们要实现函数createCurry，它能实现createCurry(f)的结果为f_curry，其中f的调用方式为f(a, b, c)，f_curry的调用方式为f_curry(a)(b)(c)。

  • createCurry的传参是一个函数f，返回结果也是一个函数f_curry
  • 返回的这个函数f_curry，当参数个数少于原函数f时，它会返回一个函数f_curry(a)、f_curry(a)(b)，当参数个数等于原函数个数时，返回原函数的执行结果。

根据以上2点，我们可以写出：

function createCurry (f) {
    const fLength = f.length
    const f_curry = function (param) {
        let args = []
        args.push(param)
        if ( fLength > args.length ) {
            // return 一个函数
        } else {
            return f.apply(this, args)
        }
    }
    return f_curry
    }

那当参数个数少于原函数的时候，return的是一个怎样的函数呢？我们看看f_curry(a)、f_curry(a)(b)，是不是其实就是createCurry(f, a)和createCurry(createCurry(f, a), b)，因此我们可以发现，这其实是一个递归调用的过程，当参数不够时，就继续返回createCurry(f, args)。 于是补充参数不够时的返回：

function createCurry (f, arr = []) {
    const fLength = f.length
    const f_curry = function (param) {
        let args = [...arr, param]
        if ( fLength > args.length ) {
            return createCurry.call(this, f, args)
        } else {
            return f.apply(this, args)
        }
    }
    return f_curry
}

验证下：

function f (a, b, c) {
    return a + b + c
}
const f_curry = createCurry(f)
console.log(f_curry(1)(2)(3))

这里我们已经实现了函数柯里化，我们再来一步步看看每步都发生了什么，以便更好的理解

  • const f_curry = createCurry(f)，初始化fLength为3， 返回f_curry

  • f_curry(1)， param为1，args为[1]，fLength为3大于arg.length，因此执行return createCurry.call(this, f, args)，最终返回一个可以访问到args为[1]的闭包函数f_curry

  • f_curry(1)(2)，同上，param为2，args为[1, 2]，fLength为3大于arg.length，因此执行return createCurry.call(this, f, args)，最终返回一个可以访问到args为[1, 2]的闭包函数f_curry

  • f_curry(1)(2)(3)，param为2，args为[1, 2, 3]，fLength为3等于arg.length，因此执行f.apply(this, args)，进入f函数， 最终返回结果6

到这里我们其实已经实现了函数柯里化，不过我们还可以再优化下。目前的createCurry的到的f_curry，无法实现f_curry(1, 2)(3)，我们稍微改进下，让它的自由度更高。

function createCurry (f, arr = []) {
    const fLength = f.length
    const f_curry = function (...params) {
        let args = [...arr,...params];
        if ( fLength > args.length ) {
            return createCurry.call(this, f, args)
        } else {
            return f.apply(this, args)
        }
    }
    return f_curry
}

验证下
```js
console.log(f_curry(1)(2)(3))// 6
console.log(f_curry(1, 2)(3))// 6
console.log(f_curry(1)(2, 3))// 6
console.log(f_curry(1, 2, 3))// 6

# 实现无限相加

# 题型

有一道很经典的题目：实现一个sum函数，具有如下功能(参数可以无限个)。

  • 题型一：
sum(1)(2)() // 3
sum(1)(2, 3)() // 6
sum(1, 2, 3, 4)() // 10
...
  • 题型二：
sum(1)(2).valueOf() // 3
sum(1)(2, 3).valueOf() // 6
sum(1, 2, 3, 4).valueOf() // 10
...
  • 题型三：
sum(1)(2) // 3
sum(1)(2, 3) // 6
sum(1, 2, 3, 4) // 10
...

以上三种题型要实现的功能基本是一样的，也就是实现一个累加函数sum，传参的形式可以任意组合。三种题型的差别就在于最终进行运算的方式不一样，分别是() 、valueOf和直接运算。

# 题解

这道题与以上我们分析的柯里化的不同是：sum的参数个数不确定。因此，我们不能再用fLength > args.length这个条件来判断是继续收集参数还是计算结果，而三种题型的差异也就是在这：如何结束参数收集而计算结果。

首先，因为参数个数不确定，我们改造下f函数

// 参数个数确定时
function f (a, b, c) {
    return a + b + c
}
// 参数个数不确定时
function f (...args) {
    return args.reduce((prev, next) => prev + next, 0)
}

# 题解一

题型一是以()来结束参数收集。仔细想想，sum(2)和sum(2)()的差别，我们可以发现结束收集参数的条件就是传参数为0。

function sum (...arr) {
    let args = arr || [] // 利用闭包保存参数
    const add = function (...params) {
        args = [...args,...params]
        if (params.length) {
            return add
        } else {
            return args.reduce((prev, next) => prev + next, 0)
        }  
    }
    return add
}

测试下

console.log(sum(1)(2)()) // 3
console.log(sum(1)(2, 3)())// 6
console.log(sum(1, 2, 3, 4)()) // 10

# 题解二

题型二是以一个方法valueOf来结束收集。那也就是结束收集参数的条件很明白的告诉我们了：执行valueOf方法时结束收集参数。

function sum (...arr) {
    let args = arr || [] // 利用闭包保存参数
    const add = function (...params) {
        args = [...args,...params]
        return add
    }
    add.valueOf = function () {
        return args.reduce((prev, next) => prev + next, 0)
    }
    return add
}

测试下

console.log(sum(1)(2).valueOf()) // 3
console.log(sum(1)(2, 3).valueOf())// 6
console.log(sum(1, 2, 3, 4).valueOf()) // 10
...

# 题解三

题型三比较特殊，没有明显的结束收集的标志。其实是利用了js执行console.log时会默认调用toString方法的原理，重写函数的toString方法，这样就可以输出我们想要的结果。

function sum (...arr) {
    let args = arr || []
    const add = function (...params) {
        args = [...args,...params]
        return add
    }
    add.toString = function () {
        return args.reduce((prev, next) => prev + next, 0)
    }
    return add
}

测试下

console.log(sum(1)(2).valueOf()) // f 3
console.log(sum(1)(2, 3).valueOf())// f 6
console.log(sum(1, 2, 3, 4).valueOf()) // f 10
...

这里要注意下，正是因为是利用了console.log会触发toString方法这一点，因此，如果不在console.log里执行sum，则不会打印出结果

function sum (...arr) {
    let args = arr || []
    const add = function (...params) {
        args = [...args,...params]
        return add
    }
    add.toString = function () {
        const res = args.reduce((prev, next) => prev + next, 0)
        console.log(res) // 在此处打印
        return res
    }
    return add
}
// 测试——不使用console.log
sum(1)(2) 
sum(1)(2, 3)
sum(1, 2, 3, 4)

则以上都不会有结果，因为没有console.log触发toString方法，toString方法里的内容不会被执行。
